版本：V2.1（2026-04-16）

目标：作为官网、PPT、销售方案、生态合作物料的统一内容底稿，快速提炼产品价值、能力边界、当前落地情况与中短期规划。

口径说明：
- 本文按“已落地能力 + 已明确规划能力”统一整理，适合宣传、售前与方案沟通。
- 涉及“规划中/预留/二期”的内容，需在对外方案中结合项目阶段说明，避免误表述为现网全部上线。

## 一句话定位
- 基于自研 IM 的企业协同与客户运营平台：以 AI 智能体为前台，以工作流、业务单据、商品与审批为中台，联动人工坐席完成从获客咨询到报价、审批、下单与售后的全链路闭环。

## 核心价值主张
- 7x24 智能服务：AI 首屏回复快，支持流式应答、知识问答、字段采集与规则转人工。
- 降本增效：AI 处理标准咨询，工作流自动执行 API、脚本、标签与卡片推送，人工聚焦高价值会话。
- 业务闭环：从会话到客户档案、业务表单、报价单、审批流、打印输出，关键过程可留痕、可复盘。
- 渠道统一：链接、二维码、嵌入页、微信/企业微信插件、独立站演示页统一接入。
- 配置优先：智能体、知识库、工作流、对接档案、价格规则、打印模板等均支持后台配置化管理。
- 多租户可商用：租户隔离、权限控制、敏感信息保护、脚本化部署，适合 SaaS 化与私有化交付。

## 能力地图（摘要）
- AI 智能体引擎
  - 通义千问（阿里百炼）接入，支持多模型切换、流式回复、客户记忆、错误回退
  - 支持知识库问答、意图识别、转人工策略、商品主数据读取开关
- 对话工作流引擎
  - 已支持 15 类节点：AI 分类、字段收集、客户标签路由、客户类型路由、入口路由、API 调用、数据库查询、脚本执行、发送卡片、转人工、结束等
  - 适合询价、留资、报价、售后、资料收集、外部系统联动等多轮业务流程
- 自研 IM 通信
  - WebSocket 实时消息，支持文本、图片、文件、系统消息与业务卡片扩展
  - 支持历史消息拉取、断线重连、弱网容错、消息 ACK 与幂等处理
- 知识与商品双知识源
  - 知识库支持 QA 对管理、上下文注入、RAG 扩展预留
  - 商品主数据可作为智能体的结构化事实来源，辅助规格说明、SKU 对比与报价前置引导
- 客户与渠道关系管理
  - CRM 客户档案、标签、互动历史、客户类型管理
  - 渠道好友与客户主档打通，兼容 ToB 联系人模式与 ToC 好友即客户模式
- 设备、账号与智能体路由
  - 支持设备、微信/企微账号、好友关系管理
  - 可根据客户标签、客户类型等条件自动路由到不同智能体
- 业务表单与业务单据
  - 支持表单定义、字段规则、模板管理、业务单据实例、AI 识别记录
  - 可与会话工作流联动，承接咨询中的资料采集、表单确认与单据生成
- 商品与价格能力
  - 支持商品扩展属性、多标识、BOM 产品结构、产品资料关联
  - 支持客户类型/标签驱动的价格规则与报价试算
- 审批与流程留痕
  - 已落地通用审批流引擎，支持发起、待办、已办、通过、拒绝、撤回
  - 已支持转办、加签、催办、超时处理，适合报价、下载权限、表单等多业务审批
- 单据打印与交付
  - 规划统一的打印模板中心，覆盖报价、订单、业务表单等单据
  - 采用结构化配置 + 服务端 HTML 转 PDF 的一致性输出路线
- 系统对接配置
  - 支持租户级第三方 API 对接档案管理，复用到工作流 `api_call`
  - 覆盖 API Key、Bearer、OAuth2、Basic、自定义签名等主流鉴权方式
- AI 质量评估与持续优化
  - 规划智能体测试方案、用例、批量执行、人工复核、自动优化闭环
- 部署与运维
  - 当前支持单机版快速部署，支持脚本化构建、上传、发布
  - 可平滑演进至多实例、集群缓存、托管数据库与异步架构

## 模块功能清单（按业务域）
- 客户端与演示入口
  - 客户态 H5 支持自动登录、历史消息拉取、AI 流式打字、图片/文件发送
  - 支持独立站演示页 `test3.html`，可跑通“商品咨询 → 留资 → 生成报价卡片”的演示闭环
- 会话与消息（M03）
  - 会话创建/复用，状态机覆盖 `ai_serving / waiting / human_serving / closed`
  - 消息类型已支持 `text / image / file / system`，卡片、语音、视频等能力可持续扩展
  - 支持离线消息、ACK 确认、消息幂等与弱网恢复
- AI 智能体（M04）
  - 智能体配置支持模型、温度、最大 tokens、系统提示词、知识库关联、工作流配置
  - 支持流式输出、客户记忆、转人工规则、商品信息读取开关
  - 问价场景可引导结合价格规则或报价流程，降低模型自由编价风险
- 工作流引擎（M06）
  - 当前后端已完整支持 15 类节点，运行时状态托管于 Redis
  - 管理端已提供可视化编辑器与 JSON 配置模式，适合复杂业务分支与外部系统调用
- 知识库（M05）
  - 已支持知识库/QA 对 CRUD、智能体上下文注入
  - RAG 检索、向量化知识增强为后续升级方向
- 入口管理与插件入口（M09 / M12）
  - 支持入口编码、二维码、链接、嵌入式入口、欢迎语、菜单按钮、智能体绑定
  - 微信/企业微信插件对接支持统一签名、OAuth、JS-SDK、会话初始化与消息链路
- CRM 与渠道好友整合（M02）
  - 客户档案、标签、互动历史、客户规则管理
  - 渠道好友能力并入客户域，支持好友绑定客户、归入企业客户、统一运营视图
- 设备/账号/智能体路由
  - 支持多设备、多账号、跨平台好友管理与客户关联
  - 支持按客户标签、客户类型等条件自动匹配智能体
- 业务表单（M14）
  - 已完成业务表单定义、字段、规则、模板、业务单据、AI 识别记录等核心能力
  - 已完成后台页面与菜单权限，支持模板 AI 分析回填、工作流联动与业务单据管理
  - 审批流完整闭环、OCR、高级模板设计器为后续增强方向
- 商品主数据与资料管理
  - 支持租户级商品扩展属性、SPU/SKU 多标识、产品结构（BOM）、产品资料关联
  - 支持属性搜索索引与产品侧资料下载入口
- 商品价格规则（M15）
  - 已支持基于客户类型、客户标签的规则化算价
  - 支持标准价/成本价作为基准，支持差额、百分比、固定价三种计价方式
  - 已提供统一算价入口与 IM 报价试算接口
- 审批流引擎
  - 已支持流程定义、发起、待办、已办、通过、拒绝、撤回、流程详情与日志留痕
  - 已支持转办、加签、催办、超时扫描，便于业务审批与审计追踪
- 报价/订单/工单（M08）
  - 已具备与会话、价格规则、报价卡片、审批流衔接的设计基础
  - 生命周期完善、单据联动与更多业务动作按迭代逐步补全
- 单据打印模板
  - 规划将报价、订单、业务表单打印统一为同一产品线
  - 主路线为结构化模板配置、统一 HTML→PDF 渲染、预览与导出一致
- 企微员工移动端（M11）
  - 已明确会话、消息中心、审批、报表、我的页等移动端范围
  - 适合企业微信工作台场景下的移动接入与客服/销售处理
- 企业微信客服接入（规划）
  - 通过企微通知推送待接入会话，客服点击消息进入 H5 服务客户
  - 与 Web 工作台能力保持一致，补齐移动办公触达链路
- AI 智能体测试（M13）
  - 规划测试方案、用例库、Excel 导入、批量执行、人工评审、自动优化新智能体
  - 适合智能体上线验收、版本回归、多智能体横向对比
- 系统对接配置（M16）
  - P0 已落地租户级对接档案管理，可供工作流通过 `integrationCode` 复用
  - 支持鉴权配置、基址管理、敏感信息保护、AI 生成配置草稿
- 多租户与权限（M01）
  - JWT 鉴权、多租户拦截器、租户上下文、菜单权限与角色能力控制
  - 共享库 + `tenant_id` 强隔离，适合多租户运营与私有化部署

接口与协议（摘要）：
- HTTP 前缀：`/api/admin`
- IM API：`/v1/im/**`（示例：`/v1/im/session/waiting`、`/v1/im/message/history`、`/v1/im/file/upload`）
- 工作流与审批：`/v1/im/workflow/**`
- WebSocket：`/api/admin/ws/im?token=JWT`
- 插件入口：`/v1/plugin/**`（示例：`/v1/plugin/entry/init`）
- 企微移动端建议接口：`/api/admin/v1/wxwork/mobile/**`
- 价格试算：`/v1/im/quotation/preview-item-prices`

## 差异化卖点（可用于宣传）
- 一体化闭环：IM + AI + 工作流 + 客户管理 + 业务表单 + 报价/审批，打通咨询到成交全过程
- 自研 IM 可控：不依赖第三方 IM SDK，消息协议、卡片能力、历史拉取、弱网策略可按业务自主演进
- 工作流可落业务：不是只有问答能力，而是可直接编排字段收集、API 调用、脚本计算、打标签、发送卡片、转人工
- 商品与价格打通：商品主数据、价格规则、报价预览、报价卡片可协同工作，更适合制造、贸易、物流类复杂询价
- 客户与渠道融合：客户主档、渠道好友、设备账号、智能体路由统一纳管，兼容 ToB 与 ToC 场景
- 对接能力强：系统对接配置覆盖主流 REST API 鉴权方式，降低外部系统接入成本
- 审批可复用：审批引擎适配报价、表单、下载权限等多业务，过程留痕、审计友好
- 轻量部署快：单机 Nginx + Spring Boot + Redis + MySQL 即可上线，适合快速试点与私有化交付

## 对接与安全摘要
- 渠道对接：微信/企微 OAuth、JS-SDK 签名；插件入口统一签名算法（HMAC-SHA256）
- 系统对接：支持 API Key、Bearer、OAuth2、Basic、自定义签名等主流鉴权模型
- 安全控制：HTTPS/WSS、全链路 Token 鉴权、敏感数据加密、多租户隔离、密钥不明文回显
- 访问边界：工作流可调用外部 API，但对接档案、密钥、Base URL 受租户与权限控制
- 错误码与签名规范详见：[17](file:///f:/Work/xm-im-erp/docs/design/17-微信企业微信插件对接接口设计.md)
