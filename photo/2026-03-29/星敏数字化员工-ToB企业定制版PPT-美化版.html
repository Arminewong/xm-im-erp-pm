基于 IM 的企业协同与客户运营平台

模块功能清单（1/2）

客户端、会话、AI 与知识库

面向 C 端体验与消息可靠性；面向 B 端可配置智能体与知识范围。

  • 客户端（H5/小程序）：自动登录、历史消息、AI 流式打字、图片/文件发送、弱网重连
  • 会话与消息：会话状态机（ai_serving / waiting / human_serving / closed）、消息幂等与 ACK、离线消息拉取
  • AI 智能体：模型参数、Prompt、知识库关联、流式输出（ai_typing）、转人工规则
  • 知识库：知识库/QA 对 CRUD、上下文注入、RAG 对接预留
体验侧首屏快、打字感真实、断线可恢复，减少「转圈」挫败感。
可靠侧幂等与 ACK 降低重复消息；状态机让「谁在服务」一目了然。
运营侧按入口绑定不同智能体；知识库可按业务线隔离扩展。

本期以 QA + 上下文注入为主；RAG 检索与更重的内容治理放在路线图二期。

安全与合规

传输、隔离与数据保护

多租户从数据模型与查询习惯上强制隔离；敏感字段单独治理。

全链路 HTTPS/WSS管理端、插件页与长连接统一加密，降低中间人风险。
多租户隔离tenant_id 强约束 + JWT 声明租户，防止串租与越权查询。
敏感数据加密手机号等字段加密存储，展示侧按需脱敏。
语音/文件合规依赖显式授权与留存策略，可按行业要求收紧。
身份与令牌JWT 短期有效 + 刷新策略；插件 OAuth 与企微签名分离配置。
审计与留痕关键业务动作建议对接现有日志平台（可按项目集成）。
上线检查清单HTTPS 强制跳转、WSS 证书、依赖版本与安全补丁纳入发版流程。
Slide 10 10 / 14
部署与运维

单机起步，可平滑演进

适合 PoC 与首期交付；业务量上升后再拆缓存、数据库与消息层。

  • 单机拓扑：Nginx + Spring Boot（8081）+ 本地 Redis + 远程 MySQL
  • 发布脚本：build → upload → deploy（systemd），可接入现有 CI
  • 演进方向：SLB + 多实例、Redis 主从/集群、RDS、MQ
现阶段单机即可跑通全链路演示与试点业务。
扩容时无状态应用水平扩展；会话粘滞或共享 Session 视架构选型。
观测建议补充 APM、日志检索与基础告警（按客户环境落地）。
备份MySQL 定期快照 + 配置文件纳管，满足恢复演练。

路线图（MVP → 二期）

分阶段交付，持续增强

先保证「聊得稳、接得住、管得了」，再增强富媒体、可视化编排与检索体验。

推荐结构：行业背景 → 痛点量化 → 上线模块 → 前后对比指标（3～6 个月）。

场景与痛点

例如：咨询高峰响应慢、报价依赖老员工、多渠道会话分散无法质检。

解决方案

效果指标

例如：首响时长 ↓、人工重复劳动 ↓、有效线索转化率 ↑、客户满意度 ↑。
